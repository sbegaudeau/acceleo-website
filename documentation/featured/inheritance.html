Inheritance: encapsulation done right

Overview

Acceleo features two major mechanisms two create a well design generator by encapsulating common logic in some modules, import and inheritance. Importing a module is easy, it gives the importing module the ability to call the public templates and queries from the imported module but inheritance can be a bit tricky. Acceleo contains two different kinds of inheritance.

Static inheritance

Static inheritance is the most used form of inheritance in Acceleo. In order to use static inheritance, an Acceleo module must declare with the keyword "extends" that it inherit from another module.

By inheriting from another module, a module can have access to all of its public and protected templates and queries. It is a very good way to hide some internal logic from other modules. The extending module can also override all the public and protected templates (not queries) of the extended module.

This behavior seems very similar to what is available in object oriented languages but major differences occurs in the dispatch strategy used by Acceleo.

Dynamic inheritance

In order to use dynamic inheritance, you not only have to create a module that extends another module with one of its template overriding one of the extended module like in the following screenshot but you also have to register your extending module with the dedicated extension point.

Dynamic inheritance is often used when a generator needs to contribute new behavior to an existing generator. You create new modules that extends the original modules and new templates that overrides the original templates. Then, in this new generator, you use the dedicated extension point as shown in the next picture to tell Acceleo that the modules contained in the selected folder should be considered as dynamic modules.

Dynamic modules are modules which are extending other modules and whose templates are overriding other templates and which are registered with the dedicated extension point. When the generation will be launched, the templates from the dynamic modules will always be used in priority other the overriden templates. You can see the behavior in the following screenshot.
