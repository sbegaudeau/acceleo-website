Overview: seemless integration within Eclipse.

The Acceleo generation module Editor supports the user with most of the features you would expect from modern IDEs: content assist, quick outline, navigation links to the declaration of model elements, template elements and variables, quick fixes, refactoring, syntax highlighting, occurences highlighting... and so much more!

Move the cursor of your mouse over the red dots to have an extended description of a specific feature.

Implementation of MOFM2T: the OMG's standard language dedicated to code generation.

MOFM2T

This template based language is an OMG's standard dedicated to code generation.

Architecture of an Acceleo module: to build your generator steps after steps.

An Acceleo module is composed of two major elements:

  • Templates, used to generate static and dynamic part of the text.
  • Queries, used to encapsulate some complex requests on the element of the models.
On the following screenshot, you can have a look at the basic elements contained in an Acceleo generator.

Templates: the core of the generator.

The template is used to generate most of the code. It can contain static area of text and dynamic parts in which expressions will use the element of the input model to compute the text to generate. Those expressions are based on a superset of the OMG's standard OCL language. Acceleo uses EMF to add new operations dedicated to the simplification of the manipulation of models.

Queries: utility operations.

Queries can be used to encapsulate an expression in order to use it at several places of your generator easily. Queries have also their result stored in cache by default (it can be changed in the preferences of Acceleo) and as such you can use a well placed query to boost the performances of your generator by en capsulating a time consuming operation in a query.

Editor

Syntax highlighting

The Acceleo editor comes with syntax highlighting for Acceleo modules. The syntax highlighting allows the user to easily identify the elements of the generator (templates and queries) and the expressions used.

The syntax highlighting of the Acceleo editor is fully customizable thanks to a dedicated preference menu.

Code completion

The Acceleo editor provides a code completion mechanism. This mechanism can be used to easily create modules, templates and queries and the different structures of the generator (for, if, let) but it also provides code completion for model elements.

The code completion inside of an expression will display the attributes and operations available on the model elements and it will also show the templates and queries that can be applied on this element.

The code completion can also be customized thanks to an extension point, for more information see the generation pattern view.

Errors detection

The Acceleo editor features a real time errors detection mechanism. Thanks to the static typing of Acceleo module, you can see the errors directly in the editor prior the launch of any generation.

This errors detection system will not only detect malformed modules with, for example, trivial mistake like missing parenthesis but it will also detect errors in the expressions. This system will help you to easily maintain your generator.

Bad practices detection

With the introduction of Acceleo 3.1, a new system has been introduced to detect bad practices in the Acceleo editor. A warning will now mark potential source of errors in Acceleo modules.

Quick fixes

The Acceleo editor contains also a mechanism to quickly fix trivial errors. Using the standard Eclipse mechanism, you can easily create a template to fix a call to an unknown operation.

Refactoring

Modern editors offer support for refactoring operations and the Acceleo editors obey to this rule with several refactoring options.

Search

Looking for something? The Acceleo editor uses the standard Eclipse search mechanism to let you find quickly all the occurrences of an element. You can easily find, for example, where a selected template is used.

Open declaration

With the open declaration mechanism, you can easily open a selected template from a template invocation expression. You can thus easily navigate in your module.

Documentation

With Acceleo 3.1, Acceleo modules can now contain documentation. This documentation is visible in the completion and in hover popup in the editor.

Code collapse

While editing a big Acceleo module, you can use the code collapsing system to mask portion of the generator in order to focus on a specific part of your module.

Quick replacement

If you start with a static block of texte, you can quickly replace several occurrences of a same piece of text by an Acceleo expression in a click.

Profiler

Profiling

When you are launching your Acceleo generator, you can use the Acceleo profiler to easily figure out the instructions used during the generation and the time spent in each of those instructions. With the profiler, you can determine the bottlenecks in your generator and then you can wisely find where your generator needs improvements.

Debugger

Debug

If you encounter a bug in your generator, you can use the Acceleo debugger to easily find your problem. Set a breakpoint, launch your generation and then move step by steps while keeping an eye on the state of all the variable in the context of the current expression to easily determine the source of your problem.

Traceability

Traceability

Acceleo contains a generation engine than can compute traceability informations. Thanks to those traceability information, you can see which model elements and which part of the generator has been used to generate a specific portion of generated text.

Perspective and views

Overrides view

The overrides view will show you all the modules in your workspace and in your plugin that you can override and it will create a new quick fix to easily override a specific template.

Result view

The result view will display all the traceability information from the last generation where the traceabiility has been activated.

Generation patterns

The generation pattern view is a view dedicated to the creation of custom quick fixes to quickly create code following some design pattern for code generators.

You can easily contribute to this view with new design pattern thanks to the dedicated extension point.

Ant, Maven, PDE

PDE Build

All Acceleo projects are Eclipse plugin projects and as such they can be easily deployed in another Eclipse instance thanks to the PDE build system. In all Acceleo projects, Acceleo maintain a build.acceleo file used by PDE to compile Acceleo modules. Thanks to this mechanism you can deploy an Acceleo generator in another Eclipse in a few clicks.

Ant

Acceleo provides the necessary Ant tasks to compile an Acceleo module and to launch a generation for all Acceleo generator.

Maven & Tycho

Acceleo also provides a Maven and Tycho based "pom.xml" to build and Acceleo project in a Maven based workflow.
